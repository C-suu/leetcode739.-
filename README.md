### （1）题目白话文解释与生僻概念说明

**题目大意：**
给定一份记录每天气温的列表，要求计算出每一天还需要等待多少天，才能迎来一个气温比当天更高的日子。如果在之后的所有日子里气温都没有再升高，那么这一天的等待天数就记为 0。

**生僻/不易理解概念解释：**

  * **下一个更大元素 (Next Greater Element)：** 这是计算机科学中常见的一类问题模型。简单来说，就是站在当前位置往后看，寻找第一个比当前位置数值还要大的元素，并计算它们之间的距离。本题中，“距离”就是天数差（索引之差）。

-----

### （2）代码解题思路解析

这段代码巧妙地使用了**单调栈 (Monotonic Stack)** 的数据结构来解决问题。

**解题思路来源：**
如果采用最暴力的解法，针对每一天都向后遍历寻找更高的温度，会产生大量的重复比较，效率极其低下。为了优化，需要一种机制把“还没找到更高气温的日子”先记录下来。

1.  依次遍历每一天的气温。
2.  只要今天的气温比“记录本”里最后记录的那天的气温低，或者一样高，说明那天的“更高气温”还没出现，那就把今天也记入“记录本”（准确地说是记录当天的**索引**，即第几天）。
3.  如果今天的气温突然升高了，比“记录本”里最后记录的那天的气温还要高，这就意味着“记录本”里最后那天终于等到了它的“更高气温”。此时，就可以算出这两天相差的天数，并把那一天从“记录本”里划掉（出栈）。
4.  继续用今天的气温去对比“记录本”里倒数第二天、倒数第三天的记录，只要今天的气温依然更高，就重复上述计算和划掉的动作，直到“记录本”里的气温都比今天高，或者记录本空了为止。
5.  最后，把今天的索引也加进“记录本”，继续往后看。

因为“记录本”（栈）里的气温总是保持着从底到顶逐渐降低的顺序，所以这种结构被称为**单调递减栈**。

### （3）代码框与逐行注释

```python
class Solution:
    def dailyTemperatures(self, temperatures: List[int]) -> List[int]:
        n = len(temperatures)                  # 获取气温列表的总天数
        ans = [0] * n                          # 初始化一个全为 0 的结果列表，长度与总天数一致
        st = []                                # 初始化一个空列表作为栈（待办事项列表），用于存放日子的索引
        for i, t in enumerate(temperatures):   # 遍历气温列表，获取当天的索引 i 和气温 t
            while st and t > temperatures[st[-1]]: # 当栈不为空，且当天气温高于栈顶索引所对应的气温时，循环执行
                j = st.pop()                   # 栈顶索引出栈（代表该天已找到更高气温），赋值给 j
                ans[j] = i - j                 # 计算等待的天数（当前天数 i 减去记录的天数 j），存入结果列表
            st.append(i)                       # 将当前天数的索引 i 压入栈中，等待后续寻找更高气温
        return ans                             # 遍历结束后，返回最终计算好的天数差列表
```

### （4）每一行代码的详细中文解释

  * `class Solution:`
    定义一个名为 `Solution` 的类，用于封装解题的方法。
  * `def dailyTemperatures(self, temperatures: List[int]) -> List[int]:`
    定义方法 `dailyTemperatures`，接收一个名为 `temperatures` 的整数列表（代表每日气温），并返回一个整数列表（代表等待天数）。
  * `n = len(temperatures)`
    计算输入列表 `temperatures` 的长度，即总共有多少天，并将其赋值给变量 `n`。
  * `ans = [0] * n`
    创建一个长度为 `n` 的列表 `ans`，并将所有元素初始值设为 0。这样做的好处是，如果某一天之后再也没有更高的气温，它在结果列表中默认就是 0，无需额外处理。
  * `st = []`
    初始化一个空的列表 `st`，在此处作为栈来使用。它用来存储那些尚未找到“下一个更高气温”的日子的**索引**（即在列表中的位置，而不是气温的具体数值）。
  * `for i, t in enumerate(temperatures):`
    使用 `enumerate` 函数遍历 `temperatures` 列表。在每次循环中，`i` 代表当前是第几天（索引，从 0 开始），`t` 代表当天的具体气温数值。
  * `while st and t > temperatures[st[-1]]:`
    这是一个 `while` 循环，执行条件有两个：一是栈 `st` 不能为空（保证 `st[-1]` 不会越界报错）；二是当前正在遍历的气温 `t`，大于栈顶元素（即 `st[-1]` 这个索引）所对应的历史气温。如果满足这两个条件，说明栈顶的那一天终于找到了比它更热的一天。
  * `j = st.pop()`
    将栈顶的元素（也就是刚好找到更高气温的那一天的索引）弹出（移除），并将其赋值给变量 `j`。
  * `ans[j] = i - j`
    计算距离：用当前找到更高气温的日子索引 `i`，减去曾经记录的日子索引 `j`，得到的就是等待的天数。将这个天数赋值给结果列表 `ans` 中的第 `j` 个位置。
  * `st.append(i)`
    在当前的 `while` 循环结束（或者根本没有进入循环）后，说明前面比当天冷的日子都处理完了。此时，需要把当天（第 `i` 天）的索引也压入栈 `st` 中，因为它也需要等待未来出现比它更热的日子。
  * `return ans`
    当整个气温列表都被遍历完毕后，所有的等待天数均已计算完成，返回最终填充好的结果列表 `ans`。

# 问题

```python
while csu and temperature > temperatures[csu[-1]]:
和
while csu and temperature > temperatures(csu[-1]):
的区别是啥？为何一个中括号一个小括号？
```

### 1. `temperatures[csu[-1]]` （使用方括号 `[]`）
* **语法含义：** 方括号代表**“索引取值”**（Indexing）。
* **具体解释：** 在当前的题目语境下，`temperatures` 是一个**列表**（List，相当于数组，里面按顺序存放着每天的温度）。`csu[-1]` 获取到的是一个整数（代表栈顶记录的某天的索引号）。使用方括号 `temperatures[...]`，相当于是在查字典或查表格，指令解释器：“去 `temperatures` 这个列表里，把第 `csu[-1]` 个位置上存放的那个温度数值拿出来”。

### 2. `temperatures(csu[-1])` （使用圆括号 `()`）
* **语法含义：** 圆括号代表**“函数调用”**（Function Calling）。
* **具体解释：** 这种写法会让 Python 解释器误以为 `temperatures` 是一个**函数**（就像 `print()` 或 `len()` 那样）。解释器会尝试把括号里的 `csu[-1]` 当作参数传递给 `temperatures` 并执行它。
* **程序后果：** 因为 `temperatures` 本质上是一个列表（数据），而不是一个可以执行的动作（函数），所以程序运行到这里会直接崩溃，并抛出错误提示：`TypeError: 'list' object is not callable`（类型错误：列表对象是不可以被调用的）。

### 总结归纳
* 只要是从**列表、数组、字典、字符串**中根据位置或键名去**获取里面的数据**，永远使用**方括号 `[]`**。
* 只要是**调用一个函数、或者实例化一个类**，才会使用**圆括号 `()`**。

# 另外一个问题

```python
while st and t > temperatures[st[-1]]: # 当栈不为空，且当天气温高于栈顶索引所对应的气温时，循环执行
  j = st.pop()                   # 栈顶索引出栈（代表该天已找到更高气温），赋值给 j
  ans[j] = i - j                 # 计算等待的天数（当前天数 i 减去记录的天数 j），存入结果列表
st.append(i)                     # 将当前天数的索引 i 压入栈中，等待后续寻找更高气温

对这个循环里面的i j还是有点晕，能不能给一个最简单的例子,一瞬间明白为何是j和ans[j] = i - j？
```

用一个最极简的例子来推演：
假设只有两天的气温：`temperatures = [20, 25]`
对应的天数索引是：第 `0` 天，第 `1` 天。

* **经历第 0 天（气温 20）：**
    因为未来还没发生，不知道啥时候会升温，只能把这天的**索引 0** 记入“待办记录本”（栈）。
    此时栈 `st = [0]`。

* **经历第 1 天（气温 25）：**
    此时代码循环里的变量 **`i = 1`**（代表今天是第 1 天）。
    开始检查“待办记录本”（进入 while 循环）：发现栈里有个 `0`，且今天的 25 度大于第 `0` 天的 20 度。说明第 `0` 天终于等到了升温的日子！

此时就到了那两行关键代码，进行“结算”：

**第一步：`j = st.pop()`**
* **动作：** 把 `0` 从待办本里划掉，并赋值给 `j`。
* **含义：** **`j` 代表的是过去一直没算出答案的那一天**。在这个例子里，`j = 0`。

**第二步：`ans[j] = i - j`**
* **`ans[j]` 的含义：** 既然是第 0 天等到了升温，那么算出来的等待天数，必须填在结果表格的第 0 个位置里，也就是 `ans[0]`。绝不能填在代表当天的 `ans[1]` 里。
* **`i - j` 的含义：** 距离怎么算？用 **今天（索引 1）** 减去 **过去那一天（索引 0）**。在这里就是 `1 - 0 = 1`。
* **整体结果：** `ans[0] = 1`。这句话完美表达了“第 0 天等了 1 天才迎来更高气温”。

---

**一句话总结这三个元素的身份：**
* **`i`** = **今天**（当前正在经历的天数索引）。
* **`j`** = **过去某天**（曾经记录在栈里，苦苦等待升温的日子）。
* **`ans[j] = i - j`** = **给过去某天填答案** = **今天** 减去 **过去某天**（算出到底熬了多少天）。
